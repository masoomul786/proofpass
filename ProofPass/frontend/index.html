STEP 2 OF 5

The Identity Bug — and the Fix

This is the core security issue that broke ProofPass's identity system. Understanding this is key to understanding why the fix matters.

⚠️ OLD BEHAVIOR (Before Fix)

The same wallet could register unlimited identities. Validation only checked if the username was taken — not if the wallet was already used:

alice.init + wallet_0x123 → register ✅ ALLOWED (ok)
bob.init + wallet_0x123 → register (SAME wallet!) ✅ ALLOWED ← 🚨 BUG
hacker.init + wallet_0x123 → register (SAME wallet!) ✅ ALLOWED ← 🚨 BUG

Result: One person = unlimited fake identities. The "digital passport" concept was broken.

🔧
The Fix: We added a second registry — wallet_registry: wallet → username. Now registration checks BOTH: is the username taken? AND is the wallet already bound? Either block = rejected.

✅ NEW BEHAVIOR (After Fix)

alice.init + wallet_0x123 → register ✅ ALLOWED (first time)
bob.init + wallet_0x123 → register (same wallet) 🚫 BLOCKED ← Fixed!
hacker.init + wallet_0x123 → register (same wallet) 🚫 BLOCKED ← Fixed!
alice.init + wallet_0x456 → register (diff wallet) 🚫 BLOCKED ← Username taken

Result: 1 wallet ↔ 1 username. True digital passport identity. Enforced at the protocol level.
